电商平台大促 ROI 优化与转化分析项目

1. 项目流程与思路业务背景：
大促期间流量成本激增，需定位高转化率的用户特征，并通过页面改版提升转化。

分析思路 (OSM模型)
O (Objective)：提升大促整体 ROI。
S (Strategy)：清洗 10 万级冗余订单数据 ，通过特征相关性分析找出高价值客群；通过 A/B 测试验证新版页面的点击率（CTR）。
M (Metric)：核心关注 CTR、转化率（CVR）和最终 ROI。

2. 遇到的问题与解决方案 
问题 1：订单数据中存在大量“极值”和“缺失值”，直接分析会导致 ROI 虚高。

解决：对缺失字段采用业务逻辑填充（如缺失来源渠道的统一标为 Unknown）；对订单金额采用 IQR（四分位距） 剔除异常大单（刷单或企业采购），确保样本代表性。

问题 2：A/B 测试结果可能受到“辛普森悖论”或随机波动影响。

解决：在 Python 中使用 statsmodels 进行双比例 Z 检验（Proportions Z-test），并设定显著性水平 $\alpha = 0.05$，确保 8.3% 的提升  在统计学上是显著的，而非随机波动。
