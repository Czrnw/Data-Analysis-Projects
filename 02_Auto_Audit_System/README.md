1. 项目流程与思路
业务背景：财务/运营每月需手工比对内部订单系统与支付宝/微信的流水，耗时长且易出错。

技术思路：基于 Pandas 构建自动化数据管道（ETL）。先对多源数据进行字段标准化，基于 order_id 进行预聚合，最后使用全外连接（Outer Join）找出差异并报警。

2. 遇到的问题与解决方案 
问题 1：浮点数精度导致“假异常”。

解决：在 Python 中，1.03 - 0.42 可能会变成 0.6100000000000001，直接比对会导致大量虚假不一致。解决方案是引入 round(x, 2) 或设置阈值（如绝对值误差 < 0.01 算作正常）。

问题 2：系统存在“一笔订单，分多次支付”的 1 对 N 情况。

解决：绝不能直接 Join。必须先对流水表按 order_id 进行 groupby(...).sum() 预聚合，将 1 对 N 降维成 1 对 1 后再关联。
